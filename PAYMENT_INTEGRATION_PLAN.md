## Payment Router Architecture

### Smart Gateway Selection
Create a payment router that automatically selects the best gateway based on:

**Selection Logic**:
```python
def select_gateway(country_code, currency, amount):
    # EU transactions → Adyen
    if country_code in EU_COUNTRIES:
        return "adyen"
    
    # India transactions → Razorpay
    if country_code == "IN" or currency == "INR":
        return "razorpay"
    
    # Default to Stripe (global coverage)
    return "stripe"
```

**Implementation File** (To be created):
- `/app/backend/services/payment_router.py`

## Database Schema

### Payment Transactions Table
```python
class PaymentTransaction(Base):
    __tablename__ = "payment_transactions"
    
    id = Column(String, primary_key=True)
    gateway = Column(String)  # "stripe", "razorpay", or "adyen"
    session_id = Column(String, unique=True)
    psp_reference = Column(String)  # Gateway-specific reference
    amount = Column(Decimal)
    currency = Column(String)
    status = Column(String)  # "pending", "completed", "failed"
    payment_status = Column(String)  # "initiated", "paid", "refunded"
    customer_email = Column(String)
    metadata = Column(JSON)
    created_at = Column(DateTime)
    updated_at = Column(DateTime)
```

### Subscriptions Table
```python
class Subscription(Base):
    __tablename__ = "subscriptions"
    
    id = Column(String, primary_key=True)
    customer_id = Column(String)
    gateway = Column(String)
    payment_method_token = Column(String)  # Stored payment method
    plan_name = Column(String)
    amount = Column(Decimal)
    currency = Column(String)
    interval = Column(String)  # "monthly", "yearly"
    status = Column(String)  # "active", "paused", "cancelled"
    next_billing_date = Column(DateTime)
    created_at = Column(DateTime)
```

## Security Considerations

### PCI Compliance
✅ All three gateways are **PCI DSS Level 1** compliant
✅ We never store raw card data (only tokenized references)
✅ Use gateway-hosted checkout pages for card collection

### Data Protection
- All payment tokens encrypted at rest
- Webhook endpoints protected with HMAC verification
- SSL/TLS required for all payment pages

### Fraud Prevention
- 3D Secure (SCA) for EU transactions via Adyen
- Stripe Radar for fraud detection
- Address verification (AVS) where supported
